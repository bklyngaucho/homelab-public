## Nginx Proxy Manager

### 🔵 Info

- [ ] **Admin UI port 81 bound to all interfaces — should be LAN-only**
  - **Why it matters:** Ports 80 and 443 legitimately need `0.0.0.0` binding since WAN traffic arrives on those ports. Port 81 (admin UI) has no reason to be reachable from outside the LAN. If a router misconfiguration or unintended port forward rule were ever added for port 81, the NPM admin panel would be directly internet-exposed. Binding to `192.168.1.33` only costs nothing and removes that risk entirely.
  - **Current:** `"81/tcp": [{"HostIp":"","HostPort":"81"}]` — bound to all interfaces
  - **Recommended:**
    ```yaml
    ports:
      - 80:80         # HTTP — all interfaces required for WAN traffic + Let's Encrypt challenges
      - 443:443       # HTTPS — all interfaces required for WAN traffic
      - 192.168.1.33:81:81   # Admin UI — LAN-only
    ```

- [ ] **No healthcheck**
  - **Why it matters:** Without a healthcheck, Docker only restarts NPM on process exit. If Nginx hangs or the Node.js management process locks up (the admin can't reach proxied services while the nginx worker keeps serving stale config), Docker keeps the container "Up" indefinitely.
  - **Current:** `Healthcheck=<nil>` — no image-level or compose-level check
  - **Verified:** Port 81 returns HTTP 200 (admin UI); port 80 returns HTTP 404 (expected nginx default for unmatched requests).
  - **Recommended:**
    ```yaml
    healthcheck:
      test: ["CMD", "curl", "-sf", "http://localhost:81"]
      interval: 60s
      timeout: 10s
      retries: 3
      start_period: 30s
    ```

- [ ] **No memory limit**
  - **Why it matters:** NPM is very lightweight (nginx + a small Node.js management process). Consistent with the pattern for all services in this stack.
  - **Current:** `Memory=0` (unbounded)
  - **Recommended:** `mem_limit: 256m`
